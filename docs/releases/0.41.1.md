# 2Hop Links Plus 0.41.1

[日本語](#日本語) | [English](#english)

## 日本語

このリリースは、カスタムビューからMarkdownノートを初めて開くときの本文内表示と、長文ノートで最上部へ戻る操作を安定させます。

- PalmWiki Homeなどのカスタムビューから同じペインでMarkdownノートを開いた場合、実際のMarkdown表示領域が準備されるまで有限回だけ待機
- Live Preview、Reading view、Source modeの現在の表示領域だけを準備完了の判定に使用し、描画と後片付けは隠れている表示モードも含めて実行
- ノート、ペイン、表示モード、本文内表示設定が変わった場合は古い待機処理を中止し、別ノートへの誤描画を防止
- 右上ボタンで最上部へ戻る滑らかなスクロールが途中で止まった場合、最大1.4秒の有限な確認で最上部へ補正
- ホイール、タッチ、ポインター、キー操作があった場合は利用者の手動スクロールを優先し、保留中の補正を中止
- PalmWiki固有の依存関係やイベント追加はなく、Obsidianの公開APIを使う他のカスタムビューにも適用

依存関係と最小対応Obsidianバージョン（1.3.5）は変更していません。

更新前に既存のプラグインフォルダをバックアップしてください。問題がある場合は、バックアップした `main.js`、`manifest.json`、`styles.css` を戻してObsidianを再読み込みできます。

## English

This release stabilizes inline rendering when a Markdown note is first opened from a custom view and makes long-note returns to the top more reliable.

- Waits for a bounded number of frames until the real Markdown host is ready when a note is opened in place from a custom view such as PalmWiki Home
- Uses only the current Live Preview, Reading, or Source host for readiness while keeping injection and cleanup consistent across hidden mode hosts
- Cancels stale readiness work when the note, leaf, mode, inline setting, or separate-pane state changes, preventing injection into the wrong note
- Rechecks an interrupted smooth return from the upper-right control for up to 1.4 seconds and corrects the final position when needed
- Cancels pending correction as soon as wheel, touch, pointer, or keyboard input shows that the user has taken over scrolling
- Keeps the compatibility fix independent of PalmWiki-specific imports, events, settings, or DOM selectors so it also applies to other views using Obsidian's public APIs

Dependencies and the minimum supported Obsidian version (1.3.5) are unchanged.

Back up the existing plugin directory before updating. To roll back, restore `main.js`, `manifest.json`, and `styles.css`, then reload Obsidian.
